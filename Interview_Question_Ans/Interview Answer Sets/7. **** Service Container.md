## **Laravel Service Container & Dependency Injection**

### **1ï¸âƒ£ What is the Service Container?**

> The Service Container is a **powerful dependency injection container** in Laravel.
> It **manages class dependencies** and **resolves objects** automatically.

* It is responsible for **binding classes or interfaces** to concrete implementations.
* Helps Laravel handle **automatic dependency injection** in controllers, services, jobs, etc.

---

### **2ï¸âƒ£ What is Dependency Injection (DI)?**

> Dependency Injection is a **design pattern** where Laravel **automatically provides the required dependencies** to a class instead of creating them manually.

```php

    private $service;
    public function __construct() {

    }

// âœ… Good â€“ DI via Service Container

```

âœ… **Benefit:**

* Loosely coupled, testable, and maintainable code.
