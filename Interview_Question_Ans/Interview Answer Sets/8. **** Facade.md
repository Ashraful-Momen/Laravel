## **What are Facades in Laravel?**

> Facades provide a **“static” interface** to classes that are available in the **Service Container**.
> They allow you to access services like `Cache`, `DB`, `Route`, or `Log` **without manually resolving them** from the container.

* Facades are **proxies** to underlying classes.
* Behind the scenes, they **resolve objects from the Service Container**.
* They **simplify code** while still being testable and maintainable.

---

## **How Facades Work**

1. You call a facade like a static class:

```php
Cache::put('key', 'value', 60);
```

2. Laravel resolves the **underlying class** from the **Service Container**.
3. Calls the requested method on the real object.
4. Returns the result.

✅ **Benefit:**

* Cleaner syntax, no `new` keyword needed.
* Access to any service bound in the container.
* Can be mocked for testing.

---

### **Example**

```php
// Using Cache Facade
Cache::put('user_1', $user, 60); // stores for 60 minutes

// Equivalent using Service Container manually
$cache = app()->make('cache');
$cache->put('user_1', $user, 60);
```

### **ASCII Diagram – How Facades Work in Laravel**

```
       ┌───────────────────────────┐
       │       Your Code           │
       │   Cache::put('key','val') │
       └─────────────┬─────────────┘
                     │
                     ▼
       ┌───────────────────────────┐
       │       Facade Class        │
       │ (Proxy / Static Interface)│
       └─────────────┬─────────────┘
                     │
                     ▼
       ┌───────────────────────────┐
       │  Service Container         │
       │  Resolves underlying class │
       └─────────────┬─────────────┘
                     │
                     ▼
       ┌───────────────────────────┐
       │  Real Class / Service      │
       │      (e.g., CacheManager) │
       └─────────────┬─────────────┘
                     │
                     ▼
       ┌───────────────────────────┐
       │     Executes Method        │
       │     put('key','value')     │
       └───────────────────────────┘
```

---

### **Short Interview Summary**

> **Facades** in Laravel are **static proxies** to classes in the Service Container.
> They allow you to call methods **statically** while Laravel resolves the underlying object automatically, keeping your code clean, readable, and testable.
