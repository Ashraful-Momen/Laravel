Here’s a **short, clear guide** on how to **set up the `Order` model in Laravel Scout** so it works with Typesense for searching.

---

## 1. Add the Scout package

If not installed yet:

```bash
composer require laravel/scout
composer require typesense/typesense-php
composer require laravel/scout-typesense-driver
```

---

## 2. Configure `.env`

```env
SCOUT_DRIVER=typesense

TYPESENSE_HOST=127.0.0.1
TYPESENSE_PORT=8108
TYPESENSE_PROTOCOL=http
TYPESENSE_API_KEY=xyz
```

---

## 3. Add `Searchable` trait to the model

```php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Laravel\Scout\Searchable;

class Order extends Model
{
    use Searchable; // ← This enables Scout

    protected $fillable = [
        'name',
        'description',
        'price',
    ];

    // Define what data is indexed in Typesense
    public function toSearchableArray()
    {
        return [
            'id'          => (string) $this->id,
            'name'        => $this->name,
            'description' => $this->description,
            'price'       => $this->price,
        ];
    }
}
```

---

## 4. Import existing orders into Typesense

```bash
php artisan scout:import "App\Models\Order"
```

This will index all current orders in Typesense.

---

## 5. Search example in Controller

```php
public function search(Request $request)
{
    $query = $request->q;

    $orders = Order::search($query)->get(); // Scout handles Typesense query

    return response()->json($orders);
}
```

---

## 6. Route

```php
Route::get('/orders/search', [OrderController::class, 'search']);
```

---

### ✅ Summary

1. Install Scout + Typesense driver
2. Add `Searchable` trait to your model
3. Define `toSearchableArray()`
4. Import existing data with `scout:import`
5. Use `Order::search($query)->get()` to fetch search results

Scout + Typesense handles indexing, searching, and returning Eloquent models automatically.
